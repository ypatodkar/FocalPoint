FocalPoint is Recursive
Self-Improvement
Every response you read trains the next one. Eye-tracking replaces human raters. The system rewrites its own instructions — no fine-tuning, no human labels.
◈ RSI Loop Implicit RLHF Antigravity Meta-Agent 6-Layer Cognitive Memory
1
What Makes This RSI
The system modifies the parameters that control its own future behavior
1
Gaze is the reward signal
WebGazer.js streams gaze coordinates. FocalPoint maps them to paragraph zones — no thumbs up/down, no ratings. Reading behavior is the implicit reward.
2
Reward updates the user profile
smooth=+1.0, confusion=−0.5, skim=−0.3, skipped=−0.2. Negative reward → complexity drops, topics flagged, format switches to bullets.
3
The system prompt is the "weights"
No fine-tuning happens. Instead, the system prompt is rebuilt from the updated profile before every Gemini call — translating profile fields into natural language instructions.
4
Every response is a different model
Gemini receives a different system prompt each turn. The same base model behaves differently because its instructions encode everything learned about this user.
5
Meta-agent closes the outer loop
At session end, Google Antigravity analyzes the full trajectory and makes higher-order corrections — catching patterns the per-turn agent misses. Writes prospective flags for next session.
6
Each session starts smarter than the last
Prospective flags are loaded into the prompt before the user says a word. The system has already pre-adapted — that's the recursive part.
2
Three Agents, Three Timescales
RSI operates at different speeds — real-time, per-turn, and per-session
⚡ Every turn · ~1ms
Prompt Agent
services/prompt_builder.py
Reads the current MongoDB user profile and rebuilds the Gemini system prompt from scratch before every single API call.
Maps profile fields to natural language: complexity_score → vocabulary level, topics_to_simplify → targeted analogy instructions, prospective_flags → proactive strategies.
The system prompt is the mechanism of self-improvement. Modifying it changes behavior without touching model weights.
# score=3 → "use very simple language" # topics=["RSI"] → "use analogies for RSI" # flags=[{action:"use_analogy"}] → # "lead with real-world analogy"
📡 Every turn · real-time
Reward Agent
services/reward.py
Receives gaze events from the previous response and computes a scalar reward — the implicit RLHF signal. Gaze replaces a human rater.
Immediately writes updates to MongoDB: complexity_score, preferred_format, topics_to_simplify, avg_words_read, reads_to_end.
The Prompt Agent reads these updates on the very next turn — the loop closes in under a second.
reward = Σ(flag_scores) / n if reward < 0: complexity ↓, format=bullets topics += [extract_topic(msg)]
🔁 Session end · RSI
Meta-Agent
services/meta_agent.py
Triggered by POST /session/end. Fetches the full session from MongoDB — every turn, every gaze event, every reward score.
Sends everything to Google Antigravity for deep pattern analysis. Catches higher-order signals the per-turn agent misses across the whole trajectory.
Writes prospective_flags like {topic:"transformers", action:"use_analogy"} to MongoDB. The Prompt Agent loads them before the next session even begins.
Antigravity → JSON analysis → update_user(profile_updates) → add_prospective_flag(flags) # next session starts pre-adapted
3
The Turn-by-Turn Loop
Six steps that execute on every single message
Step 1 · Frontend
👁️
User Reads Response
WebGazer streams raw (x, y) at ~30Hz. Dynamic EMA smoothing adapts alpha based on gaze velocity — micro-fixations get α=0.02, saccades get α=0.4.
webgazer.setGazeListener(
  (data) => onGazeUpdate(data.x, data.y)
)
// α adapts per distance
Step 2 · Frontend
🎯
Gaze → Zone Mapping
elementFromPoint(x, y) maps smoothed coordinates to data-zone paragraph spans and individual word spans (zone_0_w4).
zone_0 ← paragraph
zone_0_w4 ← word "neural"
zoneLog[zone].push(timestamp)
Step 3 · Frontend → Backend
📡
Gaze Events Sent
On next message, frontend classifies zone visit patterns into flags and sends them alongside the new message. The previous response ID links gaze to the content that was read.
{ zone: "zone_1",
  visits: 4,
  flag: "confusion" }
Step 4 · Backend · reward.py
⚡
Reward Computed
Scalar reward averaged from all gaze events. Negative reward immediately triggers profile updates — the adaptation fires before Gemini is even called.
reward = Σ(scores) / n
if reward < 0: complexity↓
if reward > 0: complexity↑
Step 5 · Backend · MongoDB
🗄️
Profile + Episode Written
User semantic profile updated in users collection. Full episode logged to episodes collection for the meta-agent to read at session end.
users["demo_user"].$set:
  complexity_score: 4
  topics_to_simplify: ["RSI"]
Step 6 · Backend · Gemini
✨
Adapted Response
System prompt rebuilt from the just-updated profile. Gemini generates a response calibrated to this user's exact reading state — different instructions from the turn before.
system_instruction =
  build_system_prompt(profile)
→ gemini-3.5-flash(contents)

Six-Layer Cognitive Memory
Modeled on human memory architecture — from milliseconds to permanent
1
Sensory Memory
Raw WebGazer stream. Millisecond lifetime. Discarded after EMA smoothing. Never stored anywhere.
{ x: 432, y: 210, t: 1719481234 }
→ smoothed → discarded
2
Working Memory
Live session state in React refs. Survives re-renders. Lost on page refresh. Chat messages persist via localStorage.
zoneLog.current = { zone_1: [t1, t2] }
sessionId.current, lastResponseId
3
Episodic Memory
Per-turn event logs in MongoDB episodes collection. Grouped by session_id. Read by the meta-agent at session end.
{ turn: 3, reward: −0.5,
  gaze_events: [...], session_id: "s_abc" }
4
Semantic Memory
Persistent user profile in MongoDB users collection. Survives across all sessions. The source of truth for the Prompt Agent.
{ complexity_score: 4, format: "bullets",
  topics_to_simplify: ["attention"] }
5
Procedural Memory
Adaptation rules hardcoded in reward.py and prompt_builder.py. The fixed logic that translates signal into behavior change. Never updated at runtime.
confusion → simplify + bullets
skipped → shorten · smooth → deepen
6
Prospective Memory
Future topic strategies written by Antigravity after pattern analysis. Applied proactively at the start of the next session — before any negative signal fires.
{ topic: "transformers",
  action: "use_analogy" } → next prompt
5
Implicit RLHF — The Reward Function
Gaze replaces human raters. No labels. No clicks. Reading behavior is the signal.
smooth
+1.0
Eyes move steadily forward through the paragraph — fixations advance, no backtracking
↑ complexity_score
maintain or shift to prose
confusion
−0.5
Same zone re-read 3+ times in short succession — the user is stuck
↓ complexity_score
switch to bullets · flag topic
skim
−0.3
Rapid back-and-forth movement within a small vertical area — scanning, not reading
↓ density
more whitespace · shorter sentences
skipped
−0.2
Zone never visited — eyes never landed here. User disengaged before reaching it.
↓ response length
front-load key information
