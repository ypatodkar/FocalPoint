Three Agents — Different Timescales
Every Turn · ~1ms
Prompt Agent
services/prompt_builder.py
▸
Reads the current user profile from MongoDB and rebuilds the system prompt before every Gemini call.
▸
Translates profile fields into natural language instructions: complexity_score → vocabulary level, topics_to_simplify → targeted analogy instructions.
▸
The system prompt is the "weights" of this system — modifying it changes the model's behavior without any fine-tuning.
if score ≤ 3: "use very simple language"
if topics: "use analogies for {topics}"
if flags: "lead with analogy for {topic}"
Every Turn · Real-time
Reward Agent
services/reward.py
▸
Computes a scalar reward from gaze events on the previous response. This is the implicit RLHF loop — gaze replaces a human clicking thumbs up/down.
▸
Immediately updates the user profile: reward < 0 → complexity drops, format switches to bullets, confused topic added to topics_to_simplify.
▸
Updates avg_words_read via EMA (0.7 × old + 0.3 × new) from zones actually visited.
reward = Σ(flag_scores) / n
if reward < 0: complexity ↓, bullets
if reward > 0: complexity ↑
Session End · RSI
Meta-Agent
services/meta_agent.py
▸
Triggered by POST /session/end. Receives the full session — every turn, gaze event, reward, and profile state — from MongoDB.
▸
Sends it to Google Antigravity (autonomous reasoning model) for deep pattern analysis across the whole session trajectory.
▸
Returns updated profile + prospective flags like {"topic":"transformers","action":"use_analogy"}. These persist to MongoDB and the Prompt Agent picks them up next session. This is RSI.
Antigravity → JSON insights
→ update_user(profile_updates)
→ add_prospective_flag(flags)
Recursive Self-Improvement — How It Works
RSI means the system modifies the parameters that control its own future behavior. The system prompt is the "weights." The AI never gets fine-tuned — instead it rewrites the instructions it gives itself.
1
User sends a message. Frontend computes gaze events from how they read the previous response and includes them in the payload.
2
Reward Agent scores the gaze: confusion zones → negative reward. Profile immediately updated: complexity drops, topic flagged.
3
Prompt Agent reads the just-updated profile and builds a new system prompt. Every single response is generated under a different set of instructions.
4
At session end, Antigravity meta-agent reviews the full trajectory. It makes higher-order corrections the per-turn agent couldn't see (e.g. complexity was rising but re-read rate was also rising — cap it).
5
Antigravity writes prospective flags to MongoDB. Next session, the Prompt Agent reads them and proactively applies the learned strategy — before any negative signal has to fire again.
6
Loop repeats. Each session, the system's self-description of the user becomes more accurate, making every future response better calibrated. That's RSI.
Six-Layer Cognitive Memory Architecture
Layer 1
Sensory Memory
Raw WebGazer stream. Millisecond lifetime. Discarded after EMA smoothing.
{ x: 432, y: 210, t: 1719481234 }
→ alpha-smoothed → discarded
Layer 2
Working Memory
Live session state in React refs. Lost on page refresh.
zoneLog.current = { zone_1: [t1,t2] }
lastResponseId, smoothedGaze
Layer 3
Episodic Memory
Per-turn logs in MongoDB episodes collection. Input to the meta-agent.
{ turn: 3, reward: -0.5,
  gaze_events: [...],
  session_id: "s_abc" }
Layer 4
Semantic Memory
Persistent user profile in MongoDB users collection. Survives across all sessions.
{ complexity_score: 4,
  topics_to_simplify: ["RSI"],
  avg_words_read: 180 }
Layer 5
Procedural Memory
Adaptation rules hardcoded in reward.py + prompt_builder.py. Never changes at runtime.
confusion → simplify + bullets
skipped → shorten
smooth → increase depth
Layer 6
Prospective Memory
Future topic strategies written by Antigravity. Applied proactively in the next session.
{ topic: "transformers",
  action: "use_analogy" }
→ injected into next prompt
Implicit RLHF — Reward Function
smooth
+1.0
Eyes move steadily through the paragraph start to finish
↑ complexity_score · maintain format
confusion
−0.5
Same zone re-read 3+ times in a short window
↓ complexity · switch to bullets · flag topic
skim
−0.3
Rapid back-and-forth movement within a small area
↓ density · more whitespace · shorter sentences
skipped
−0.2
Zone never visited — user's eyes never landed here
↓ response length · front-load key info
